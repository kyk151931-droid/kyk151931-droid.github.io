251229 보조강사님 깃허브 수업 정리

1. 깃허브(원격)에서 생성 후 복제(Clone)하는 방식
이미 온라인에 있는 방을 내 컴퓨터로 가져오는 것

주소 복제 및 폴더 추가 과정: VS Code의 'Git 리포지토리 복제' 기능을 쓰면 주소를 직접 복사하거나 미리 폴더를 추가할 필요가 없다. VS Code가 알아서 연결해 주기 때문.

fetch/pull/push 아이콘 클릭: 보통 VS Code 하단의 '변경 내용 동기화(Sync Changes)' 버튼 하나로 해결. 내부적으로는 push와 pull을 동시에 수행.


프로세스
GitHub: New Repository 생성

VS Code: Git 리포지토리 복제(Clone) > GitHub에서 복제 선택 > 리포지토리 선택.

Local: 로컬 폴더 지정 후 열기.

작업: 파일 수정 및 생성.

반영: * 커밋(Commit): 좌측 3번째 아이콘(소스 제어) > 메시지 입력 > '커밋' 버튼 클릭.

푸시(Push): '변경 내용 동기화' 클릭하여 GitHub 사이트에 반영.


2. 로컬(내 컴퓨터)에서 시작하여 연동하는 방식
내 방에 있는 일기장을 온라인 보관함에 올리는 것.

게시 Branch: 로컬에만 있는 코드를 GitHub에 '새로운 저장소를 만들면서 동시에 올릴 때' 아주 유용.

주의사항 (중요): 파일이 하나도 없으면 커밋을 할 수 없다. (빈 폴더는 인식하지 않음)


프로세스
Local: 폴더 생성 > VS Code로 열기.

파일 생성: 아무 파일(예: index.html)이나 하나 생성.

초기화: 소스 제어 탭 > 리포지토리 초기화(Initialize Repository) 클릭.

커밋: 메시지 입력 후 커밋.

게시: Branch 게시(Publish Branch) 클릭 > GitHub(Public/Private) 선택.
